# 创作者后台 UI 改版记录

## 改版目标

将创作者后台从基础管理界面升级为面向内容生产流程的 Creator OS：以工作台、AI 创作、作品管理、素材库、Prompt 模板和内容审核为主链路，统一视觉语言，并补齐移动端使用体验。

## 本次同步范围

- 从改版副本 `/Users/beststar/Documents/Codex/bytedance-train` 精确同步前端 UI 增量。
- 保留当前项目已有的 API、认证、内容流、服务端和数据库未提交改动。
- 未覆盖当前项目的 `pnpm-lock.yaml`，避免破坏现有依赖状态。
- 新增内容审核页并接入现有 `/api/content/works` 数据接口。

## 视觉系统

- 品牌主色：头条红 `#f5222d`，用于主操作、当前导航和关键状态。
- 中性背景：冷灰工作区背景配合白色内容卡片，降低长时间创作的视觉负担。
- 组件语言：统一 12–16px 圆角、轻量边框、低对比阴影和红色键盘焦点环。
- 字体：优先使用苹方、冬青黑体和微软雅黑，适配中文后台的信息密度。
- 动效：页面进入使用一次性轻位移动画，并遵循 `prefers-reduced-motion`。

## 页面改动

### 工作台

- 新增创作流水线、创作入口、数据概览、审核状态和内容趋势等模块。
- 工作台入口统一为 `dashboard`，数据中心和创作灵感复用数据视图。

### AI 创作

- 重构为空状态创作台与对话创作台两种界面。
- 增加创作记录、自动保存、安全检查、草稿操作和 Prompt 快速入口。
- 移动端隐藏内部历史侧栏，模板卡片由多列降为单列，输入区压缩内边距。

### 内容审核

- 新增待审核、审核通过、需要处理、平均质量分四项概览。
- 支持作品标题与正文搜索，展示安全状态、质量评分和更新时间。
- 平均质量分按所有已评分作品实时计算。
- 桌面端采用结构化表格，移动端切换为双列卡片行，避免横向溢出。

### 全局响应式

- 桌面端保留固定侧栏；移动端新增五入口底部导航。
- 顶部搜索栏改为弹性宽度，窄屏隐藏辅助按钮和快捷键提示。
- 主内容区预留移动底部导航安全空间，并适配 `100dvh`。

## 工程质量

- TypeScript：`pnpm exec tsc --noEmit` 通过。
- 生产构建：`pnpm build` 通过，首页成功静态预渲染。
